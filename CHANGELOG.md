## [Unreleased]

> Running log for the hard fork. The extension's patch version (currently 0.2.x) is bumped on nearly every build so a new VSIX can be installed and reloaded during development — it is a build counter, not a semantic release. Everything below has shipped in those builds; it is grouped here by change type until the fork cuts its first tagged release.

### Added

- **Gutter markers are icons now**: every block's gutter grabber shows the same icon its slash-menu row uses — pilcrow for paragraphs, the bullet/ordered/task list icons for list items, quote, code, diagram, math, image, and HTML icons — instead of source glyphs (`-`, `![]`, `` ``` ``…); headings show the slash menu's `H1`–`H6` badge instead of hashes (the sticky heading bar matches). All markers are now a uniform size, so gutter columns line up; the drag pill names the block type ("Paragraph", "3 blocks") instead of echoing a glyph. Also new: **footnote definitions** get a gutter marker and block menu, and a **Mermaid/math block's marker stays visible while its preview is showing** (it used to vanish with the collapsed code view). Fixed: task-list grabbers were pushed ~20px too far into the margin, out of column with other list items.
- **Block selections are real block selections**: selecting whole blocks (via Escape, the marquee, Shift+↑/↓, or after a multi-block drag) now uses a first-class block-range selection instead of a stretched text selection. Every block participates — horizontal rules, images, and empty paragraphs included, which a text selection silently skipped — the native blue text highlight is suppressed in favor of the block tint, copy still yields the whole blocks as markdown, typing or Backspace replaces/deletes exactly the selected blocks, and undo restores both the blocks and the selection.
- **Block keyboard shortcuts**: the block layer is now fully keyboard-reachable. **Escape** escalates any caret or text selection to selecting its whole block (works in empty paragraphs too; press again to collapse back); **Shift+↑/↓** grows or shrinks a block selection one block at a time honoring the anchor — a single selected block pressed the other way flips direction and keeps itself selected — and these keys never touch a normal text selection. **Cmd+A** climbs a ladder: the block's text → the block itself → every block (tables and code blocks keep their native select-all). **Alt+↑/↓** (or **⌘⇧↑/↓**) moves the current block — or every selected block, and the moved run stays selected — through the same machinery as the gutter drag: an unselected heading carries its whole section, fold state travels, the destination flashes, and one move is one undo step.
- **Marquee block selection**: click and drag in the editor's margins (left of the content, or right of it) to rubber-band-select whole blocks — the rectangle draws in the accent color, covered blocks tint live, and releasing selects them all, ready to drag by any covered gutter marker. The window auto-scrolls when the rubber-band nears the viewport edges (the rectangle stays anchored to the content, so scrolling extends it), leaf-only runs like a lone horizontal rule select for real, Escape cancels, and pointer-downs inside text never turn into a marquee (your text selection is never stolen).
- **Every list item is its own block**: list items — at any nesting depth — now each get their own gutter grabber with a source-true glyph (`-`, the item's actual number like `3.`, `[ ]`/`[x]` for tasks) instead of one marker for the whole list. Drag an item to reorder it within its list or refile it into another list (its nested children travel with it; the drop line indents to the target list's depth; a list dissolves cleanly when its last item leaves). An item's menu offers Move Up/Down among its siblings, Duplicate, Delete, and Copy as Markdown (copied in its list flavor, e.g. `1. text`), plus the whole-list conversions under a "Turn list into" header — so bullet ↔ ordered ↔ task stays one click away.
- **Drag blocks to reorder them**: every gutter marker is now also a drag handle — press and drag any block's glyph (`P`, `#`–`######`, `-`, `>`, ` ``` `, `![]`, …) to move the block, with an accent drop line snapping to the nearest position between blocks. **Headings drag their entire section**: `## Setup` brings everything under it until the next same-or-higher heading, collapsed or not. The window auto-scrolls when dragging near the viewport edges, Escape cancels a drag in flight, one drag is one undo step, and a plain click still opens the block menu (the marker tooltip now reads "Click for options · Drag to move").
- **Every block gets a gutter marker**: lists (`-`, `1.`, `[ ]`), blockquotes (`>`), callouts (`[!]`), code blocks (` ``` `), standalone images (`![]`), and raw HTML blocks (`<>`) now show a hover-revealed gutter glyph mirroring their markdown source — the same treatment paragraphs' `P` and headings' `#` already had — and clicking it opens the same block menu. Conversions adapt to the block: lists retype between bullet/ordered/task in place or unwrap to paragraphs/headings (one heading per item), quotes and callouts convert to each other in place or unwrap, anything can become a code block (its literal markdown source goes inside the fence), and image/HTML blocks get an actions-only menu (no "turn into" that would mangle them). Code blocks convert only via their own fence for now — turning code back into rich blocks needs the per-block source editing planned separately. Tables keep their own richer controls; horizontal rules don't yet have a marker.
- **Gutter block menu — turn into + block actions**: clicking a block's gutter marker (`#`–`######` on headings, `P` on paragraphs) now opens a full block menu instead of the level-only switcher. The **Turn into** section converts the block in place — Paragraph, H1–H6, Bullet/Ordered/Task List, Blockquote, Callout, or Code Block (converting to a code block preserves the block's literal markdown source inside the fence, so nothing is lost). Below it, **block actions**: Duplicate, Copy as Markdown, Move Up / Move Down, and Delete; headings also get **Copy Link** (a `[title](#anchor)` markdown link) and their moves operate on the whole section — `## Setup` moves with everything under it, hopping over the neighboring section in one step, and never interleaving into it. Move rows grey out at the document edges. The markers' tooltip is now "Block options".
- **Table column alignment**: right-click any cell → **Align Column Left / Center / Right** sets the GFM alignment markers (`:---` / `:---:` / `---:`) for that column — header and body cells render the alignment immediately, and the markers round-trip byte-faithfully. Re-picking the column's current alignment clears it back to the unmarked `---`. A multi-column cell selection aligns every spanned column. Also fixed: inserting a column no longer silently writes an explicit `:---` (left) marker the user never chose — new columns are born unmarked.
- **Paragraph gutter — turn a paragraph into a heading from the margin**: hovering a top-level paragraph reveals a "P" in the gutter at the same resting contrast as the heading `#` markers; hovering or focusing the marker itself brings it to their full hover treatment, and clicking it opens the same P / H1–H6 menu — so paragraph → heading is as reachable as heading → paragraph. The hover zone bridges the gap between text and marker, so mousing over to click never loses it. Paragraphs inside lists, quotes, and tables are untouched.
- **Code toolbar dropdown (code block + Mermaid + math block)**: the Code Block button is now a **Code** dropdown — a plain code block on top, then **Mermaid Diagram** and **Math Block** below a divider (both are fenced code blocks with the language preset). Mirrors the Quote and Lists dropdowns; all three remain in the slash menu too.
- **Quote toolbar dropdown (blockquote + callouts merged)**: the Blockquote button and the Callouts dropdown are now one **Quote** dropdown — a plain blockquote on top, then the five GitHub callout types (Note / Tip / Important / Warning / Caution) below a divider. It ships on the visible bar where the Blockquote button used to, so the callout types are now one click away by default instead of hidden. Place or hide it via `markdownWysiwyg.toolbar.items.quote` (the old `toolbar.items.blockquote` / `callouts` settings are removed). Blockquote and each callout type remain in the slash menu and command palette.
- **Lists toolbar dropdown**: the Bullet List, Ordered List, and Task List buttons are now a single **Lists** dropdown that highlights (with a filled accent row) whichever list type the cursor is in, so three toolbar slots collapse into one. Place or hide it via `markdownWysiwyg.toolbar.items.listMenu`; the old per-list `toolbar.items.bulletList` / `orderedList` / `taskList` settings are removed (each list type is still one click away from the slash menu, the command palette, and its `-` / `1.` / `- [ ] ` input rule). The slash menu still lists all three as separate rows.
- **Change a heading's level from the gutter**: clicking a heading's `#`/`##` gutter marker opens a small menu to retype it — **Paragraph** or **H1–H6**, with the current level checkmarked — without dropping to raw Markdown. The marker sits beside the collapse chevron as a distinct target (they never overlap) and works on every heading, foldable or not.
- **Master Proofreading switch — gate the whole feature on or off**: a **Proofreading** on/off switch at the top of the Checks toolbar menu (and a matching **Toggle Proofreading** command, ⌘⌥⇧D / Ctrl+Alt+Shift+D) turns spelling, grammar, and style checking on or off all at once for a distraction-free drafting pass. It's a true gate: flipping it off hides the rest of the menu and stops all scanning, and flipping it back on restores **exactly** the checks you had enabled — it never rewrites your individual choices. When it's off, the Checks toolbar button dims so you can tell proofreading is disabled at a glance instead of mistaking an unchecked document for a clean one. Every row in the menu — the master, the three domain toggles, and each style sub-check — is now an on/off switch. Backed by a new `markdownWysiwyg.proofreading.enabled` setting.
- **Proofreading — style checks, spell check, and grammar**: an offline writing-assist layer underlines inline issues; click one for a popup that applies a suggested fix, adds the word to your dictionary (spelling, persisted to settings), or ignores the finding for the session. Style checks flag fillers, redundancies, clichés, wordiness, AI-tell vocabulary and artifacts, passive voice, negative parallelism, over-long sentences, rule-of-three runs, em-dash overuse, and non-ASCII punctuation; a Harper-backed grammar engine and an offline spell checker run alongside. Everything lives under a unified **Checks** toolbar menu, gated by the master `markdownWysiwyg.proofreading.enabled` and individually toggleable via `markdownWysiwyg.styleCheck.*` / `spellCheck.enabled` / `grammarCheck.enabled` settings — every one now defaults on, including passive voice (`styleCheck.passive`) and negative parallelism (`styleCheck.negativeParallelism`), with the master **Proofreading** switch as the escape hatch when they get noisy — and per-document exceptions and a personal word list suppress false positives. Checks don't fire inside code, math, and other spans where the rules don't apply.
- **Callouts / admonitions**: GitHub alerts (`> [!NOTE]`, `[!TIP]`, `[!IMPORTANT]`, `[!WARNING]`, `[!CAUTION]`) and Obsidian-style callouts (`> [!tip]- Optional title`, aliases like `hint`/`faq`/`error`, `+`/`-` fold markers) render with a per-kind icon and accent color, a collapsible body (visual only — folding never rewrites the file), and a type-picker dropdown in the title bar. Insert via the slash menu ("Callout"), the command palette, or by typing `[!note] ` at the start of a blockquote. Unknown types render neutrally; the marker line's exact source bytes round-trip verbatim, and a marker line with inline formatting deliberately stays a plain blockquote (its exact bytes couldn't be guaranteed on save, so it's left untouched).
- **Delete a callout in place**: callouts now have a delete button (a trash icon in the callout header, revealed on hover / keyboard focus) so you can remove one without dropping to raw Markdown — matching how images are deleted.
- **Callouts toolbar dropdown & typed slash inserts**: a **Callouts** dropdown button (left group by default; `markdownWysiwyg.toolbar.items.callouts`) inserts a Note / Tip / Important / Warning / Caution callout directly, each with its own icon. The slash menu now surfaces those five GitHub types by name too — typing `warning`, `callout`, or an Obsidian alias like `attention` reveals the specific type alongside the generic "Callout" row (the five type rows are search-only, so the browse list stays uncluttered).
- **Container directives**: `:::name` fenced blocks (the Docusaurus admonition syntax — titles, `{attrs}` kept raw, `::::` nesting) render as labeled containers with an editable body; known names (note/tip/info/warning/danger/…) pick up callout-style accents. Typing `:::name ` in an empty paragraph creates one. Only genuine `:::`-fenced blocks are recognized — a `:word` in ordinary prose is never mistaken for a directive. Unclosed or formatted fences stay plain paragraphs.
- **Highlight**: `==marked text==` (Obsidian) renders as a theme-aware highlight; typing `==text==` applies it live, a Highlight command sits in the palette, and an opt-in toolbar button ships hidden by default (like Footnote). Strict grammar — no `=` inside, no edge spaces — anything else stays plain text, byte-preserved.
- **Editable callout and directive titles**: the title in a callout's or directive's header is an inline text field — click to edit, Enter or click away saves, Escape reverts, and an untouched blur never dirties the file. Callout titles write back backslash-escaped so a typed `*x*` can't downgrade the callout to a plain blockquote on reload; directive titles are sanitized (fence lines can't carry escapes) with any `{attrs}` block preserved. ⌘A/Ctrl+A inside a title selects the title only.
- **Callout icon affordances**: the kind icon is now the picker button — accent-chip hover, keyboard focus ring, and full keyboard menu support (Enter/Space/ArrowDown opens, arrows navigate, Enter applies, Escape closes) with `aria-haspopup`/`aria-expanded`/menu roles.
- **Notion callouts**: `<aside>` blocks from Notion's "Export as Markdown & CSV" (the shape Notion documents as having "no Markdown equivalent") render as editable callouts — the leading emoji becomes the icon and accent color (💡 tip, ⚠️ warning, 🐛 bug, …), the body is real editable markdown (the raw first segment is sub-parsed, so `**bold**` inside is a live mark, not literal text), and the exact byte shape round-trips including the blank-line-before-`</aside>` variant. Out-of-grammar shapes (the `<img>`-icon variant, unclosed asides, a blank line straight after `<aside>`) stay as the read-only sanitized HTML preview, byte-preserved.
- **Keyboard focus indicators**: every button and select in the editor UI now shows a `focusBorder` outline when focused via keyboard (mouse clicks stay ring-free) — previously keyboard focus was invisible across the find bar, toolbar, and link popup.
- **Find toggle shortcuts**: Alt+C / Alt+W / Alt+R (⌘⌥C/W/R on macOS) toggle Match Case / Whole Word / Regex from anywhere in the find bar, as in VS Code; the toggle tooltips show the shortcut.
- **Tooltips on keyboard focus**: tabbing onto a button now shows its tooltip (including shortcut hints), hidden again on blur or Escape; click focus stays silent since mouse users already have hover.
- **Keyboard activation for toolbar controls**: Enter/Space now activates toolbar buttons (previously they were wired to mousedown only, so keyboard focus could reach them but not trigger them), and the toolbar dropdowns (Format, Font, Checks, Settings, overflow) open from the keyboard — Enter/Space or ArrowDown on the trigger, arrows to move across rows, Enter to apply, Escape or tabbing away to close. Triggers now carry `aria-haspopup`/`aria-expanded`.
- **Slash command menu**: typing `/` at the start of a block (or after a space) opens a filterable, keyboard-first insert menu with markdown-shortcut hints per row. The browsable list is grouped into **Text** (paragraph, headings), **Lists**, and **Insert** (table, image, code block, blockquote, callout, mermaid diagram, inline math, math block, link, footnote, horizontal rule). A **Show all commands** button in the footer reveals the search-only rows — inline formatting, view controls (fonts, font size, table-of-contents, toolbar), and app actions (find, checks, settings) — which are otherwise surfaced by typing. Type to filter, arrows to navigate, Enter/Tab or click to insert, Escape to dismiss (typed text stays). The menu is context-aware (same-type list/quote toggles and items that can't be inserted at the cursor are hidden — inside a table cell, only inline items appear), never opens from undo/paste/external rewrites, and never steals focus. Every row runs the same editor command as the toolbar and command palette, so behavior is identical everywhere.
- **Inline math vs math block**: the slash menu now distinguishes **Inline Math** (`$…$`, renamed from the ambiguous "Math") from **Math Block** (a display `$$…$$` block — previously only reachable by typing `$$␣`).
- **Font, proofreading, toolbar, and table-of-contents commands in the Command Palette**: the font presets (Editor / Sans-Serif / Serif / Monospace), Increase / Decrease Font Size, the Check Spelling / Grammar / Style toggles, Toggle Toolbar, Toggle Table of Contents, and Swap Table of Contents Side are now first-class commands in the Command Palette (Cmd/Ctrl+Shift+P) — previously several of these (notably font sizing) were reachable only from the toolbar or slash menu. Toolbar and TOC visibility/side use a single state-aware toggle (the slash-menu row's label reads "Show"/"Hide" or "Move Left"/"Move Right" to match the current state) rather than a redundant pair of show/hide entries. They remain in the slash menu too.
- **Smart link resolution** (`markdownWysiwyg.smartLinks`, default on): clicking a local link resolves it the way a site generator would publish it — workspace-root paths (`/docs/guide`, VS Code's own convention), ancestor content roots (inside a Hugo `content/` tree, `/write/uber` opens `content/write/uber/index.md`), `.md`/`.markdown`/`index.md`/`_index.md` suffix inference, percent-decoding, and a workspace-wide suffix-match fallback. A miss shows a non-modal warning instead of VS Code's open error; resolution is click-time only (zero rendering cost). Off: links resolve relative to the document — a leading `/` still means the workspace root, never the filesystem root (previously it resolved to the filesystem root and always failed).
- **Wikilinks**: `[[target]]`, `[[target|alias]]`, `[[target#heading]]` (Obsidian conventions) parse, render as links, and round-trip **byte-identically** (the exact source bytes between the brackets are preserved on save). Clicking resolves bare names by filename across the workspace (case-insensitive, markdown files preferred, shortest path wins); `[[#heading]]` jumps in page. Typing `[[` opens bare-name autocompletion (a bundle's `index.md` completes as its directory name; duplicate names disambiguate as paths). Parsing is always on so round-trip behavior never depends on configuration; navigation and autocomplete sit behind `smartLinks`. Note: where a `[a]: url` definition exists, `[[a]]` now reads as a wikilink rather than a bracketed shortcut reference — serialization is unchanged either way. `[[x]](url)` stays a normal CommonMark link (the citation pattern), never a wikilink. A bare `[[` opens the suggestion menu immediately; `\|` in a target reads as a plain pipe (Obsidian's in-table spelling), and newly created alias wikilinks escape their pipe inside table cells so a cell can never split.
- **Resolved-target hint**: the link popup shows where a local link will actually open (`→ content/write/replit/index.md`) — the same resolver the click uses, updating live under the URL input while editing; a smart-mode miss reads "not found in workspace".
- **Link edits save on blur**: the popup's confirm button is gone — Enter applies and closes, and moving focus away applies the edit while keeping the panel open, so a change is never lost to a stray click.
- **Link popup, redesigned interactions**: clicking a link now opens its popup and keeps it open (pinned) until you click away or press Escape — previously the popup only appeared on hover and vanished the moment you clicked into the link. The header actions are `open · copy · unlink · edit` with uniform spacing: **Copy link** copies the URL; **unlink** is a clear slashed-link icon (remove the link, keep the text) replacing the old bottom-of-panel trash can; unlink is hidden for read-only links (reference / same-page wikilinks) and while creating a new link, where the redundant Edit toggle is now hidden too. The **Format** control (Markdown / `[[wiki]]`) only appears when a wikilink is actually possible — it's hidden for external URLs and anchors instead of showing a greyed-out option, is labeled **Local link format**, and its choices are named with an example shape (`Markdown — [text](url)`, `Wikilink — [[page]]`).
- **Link format switch**: the link popup's edit panel and the toolbar's insert-link prompt carry a two-option format control (`markdown` / `[[wiki]]`) — standard markdown is the default for new links, an existing link opens on its own current format, and switching converts the link in place in both directions. The wikilink option disables for external targets.
- **Cross-file heading navigation**: `file.md#some-heading` and `[[page#Heading]]` open the target and scroll to the matching heading, using the same slugs as in-page anchors.
- **Consistent right-click menus**: the toolbar now has its own context menu (Customize Toolbar / Extension Settings / Keyboard Shortcuts, mirroring the settings gear); Copy as HTML / Copy as Markdown appear in table and link menus too, and with nothing selected they copy the block under the cursor (right-click a table → copy the whole table); every content menu gains **Edit Raw Markdown** in a bottom group, preserving the scroll position like the toolbar button; the table menu shows a divider between the insert and delete actions.
- **Frontmatter chip editing**: list-valued YAML keys in the frontmatter panel edit as removable chips instead of raw text, with autocompletion from values already used elsewhere in the workspace; non-flat YAML is left untouched rather than flattened. The panel has a collapse toggle, and `markdownWysiwyg.frontmatterExpanded` sets whether it starts open.
- **Frontmatter panel alignment**: the field-name labels now line up with the document's body-text column instead of being indented to make room for the delete button, and each row's delete button sits out in the left gutter — aligned with the heading gutter markers (like `#H1`) — revealed on row hover.
- **Customizable toolbar**: every toolbar item can be individually shown or hidden and reordered (`markdownWysiwyg.toolbar.items.*`, `toolbar.order`), and the whole toolbar can be hidden (`toolbar.visible`) for a chrome-free writing surface — with slash-menu parity so hidden actions stay reachable. The shipped default layout keeps editing controls on the left and utilities on the right. In customize mode (Settings → Customize Toolbar), hovering any item chip — in the toolbar zones or the "Hidden — drag to add" tray — now shows its name, so you can tell items apart before dragging.
- **Font presets and size**: a Font menu switches between sans / serif / mono presets, each with its own customizable stack (`markdownWysiwyg.fontPreset`, `fontFamilySans` / `fontFamilySerif` / `fontFamilyMono`), and a content font-size stepper (`fontSize`) adjusts the editor body independently of the UI.
- **Content width control**: a **Full Width / Fixed** segmented control in the typography (**A**) menu, backed by `markdownWysiwyg.contentWidth`. **Full Width** (default) fills the pane as before; **Fixed** caps and centers the content at `markdownWysiwyg.maxContentWidth` (in `ch`, default 100), so the measure scales with the content font size. The choice applies live to every open editor.
- **Resizable Table of Contents**: the TOC docks on the right by default (`markdownWysiwyg.tocPosition`) and its width is drag-adjustable, with a double-click to reset. The drawer is chrome-free — it blends with the editor background (no border or panel shade), and two controls float in its top corner over the scrolling heading list: a hide button (VS Code side-bar glyph) that collapses the drawer to a small reveal tab pinned at the same corner, and a side-switch button (⇄) that moves the drawer to the other edge in place — persisting `tocPosition`, so changing sides no longer needs the Settings UI. Hovering the drawer reveals a faint edge border, and hovering that edge (or dragging it) brightens it to the theme accent — matching VS Code's sash — with a forgiving hit area.
- **Rebindable keyboard shortcuts**: all UI-level and find-navigation shortcuts moved to VS Code's keybinding system, so every one is user-rebindable and discoverable in the Keyboard Shortcuts editor.

### Changed

- **The block menu now matches the slash menu's presentation**: every Turn-into row carries the same icon or H1–H6 badge, the full name ("Paragraph", "Heading 2" — not "P"/"H2"), and the literal markdown hint (`##`, `-`, ` ``` `) right-aligned, all drawn from the same source as the slash menu so the two can never drift. Rows sit under labeled **Turn into** / **Actions** headers, action rows have icons too (duplicate, copy, link, move, trash), and the block's current type shows as a filled accent row — the same treatment as the toolbar's text-style dropdown. While the menu is open, the block it targets carries a quiet background tint so it's obvious what Duplicate or Delete will hit.
- **Gutter markers stay quiet while you type**: any keystroke hides the hover-revealed block markers until the mouse moves again, so the gutter never flickers alongside the caret. Typing is also faster in long documents — the gutter no longer rebuilds its buttons on every keystroke.
- **Dragging shows what's moving**: a small pill rides the cursor during a block drag with the block's glyph — and a block count when several blocks move together (e.g. `#  5 blocks`) — plus a dimmed "esc to cancel" hint. Everything included in the drag dims, so the moving run is visible at a glance; the same dimming appears the moment a selection spans multiple blocks (one visual language for "these blocks are included", shared with the future marquee). While dragging, no other hover chrome pops — tooltips, link popups, and marker reveals stay quiet until the drop. After a drop, a brief accent flash marks where the moved blocks landed. You can also drop a block right back into its own slot: hovering near its origin hides the indicator and releases cleanly in place.
- **Tooltips and the drag pill are inverted chips**: both now render with reversed contrast in every theme — dark chip with light text on light themes, light chip on dark — built from the editor's own foreground/background tokens, so the two always match and always stand out.
- **List item markers sit clear of the bullets and reveal one at a time**: an item's glyph no longer overlaps its bullet, and hovering an item reveals only that item's marker — not its nested children's.
- **Drag several blocks at once**: select text spanning multiple blocks, then drag the gutter marker of any block inside the selection — the whole covered run moves as one (one drop, one undo step), with the pill counting the blocks. While such a selection exists, every covered block's gutter marker surfaces and the covered range shows a block-scope tint in the editor's own selection color (dimming is reserved for drags — selection now reads as selection), and after the drop the moved run stays selected — ready to grab again.
- **Undo and redo stay where you were**: undoing or redoing a gutter-menu conversion, move, or drag now restores the caret (and scroll position) at the block that was acted on — it used to snap back to wherever the caret last sat, often the top of the document. Copy actions never move your caret or selection.
- **Gutter markers are easier to click**: the `#`–`######` and `P` gutter markers now have a much larger click target and hover background (padded well beyond the glyph) while the characters themselves render exactly where they did. The `P` marker's tooltip now says "Change heading level" (matching the heading markers — it's the same P / H1–H6 menu) instead of "Change text style".
- **Inline math is now edited in place, like inline code**: arrow keys walk *into* a `$…$` formula character by character — the rendered KaTeX swaps to the raw LaTeX source (with dimmed `$` delimiters) while the caret is inside, and re-renders the moment it leaves. Clicking a formula or backspacing against its edge also reveals the source (no more click popover). Typing `$` at the end of the source closes the formula; deleting all its characters removes it. Shift+Arrow extends a selection through a formula one character at a time instead of jumping past it whole. Moving the caret through a formula never modifies the document, and formulas round-trip byte-identically exactly as before.
- **Footnote and Horizontal Rule buttons reflect the caret too**: the Footnote toolbar button lights up while the caret is inside a footnote definition (or a footnote reference is selected), and the Horizontal Rule button lights when a rule is selected; a selected reference-style image (`![alt][id]`) lights the Image button. The bar also goes neutral while typing in an image caption, matching the callout-title behavior.
- **The toolbar now reflects what the cursor is in**: inline-formatting buttons (Bold, Italic, Strikethrough, Highlight, Inline Code, Link) light up when the caret or selection carries that mark; the Link button also lights on a selected `[[wikilink]]`, the Inline Math button on a selected `$…$` formula, and the Image button on a selected image; the Table button and the Lists / Quote / Code dropdown triggers light up when the caret is inside a table / list / blockquote / callout / code block; and inside those dropdowns the exact row you're in (bullet vs ordered vs task, blockquote vs the specific callout kind, code vs Mermaid vs Math) is filled. The always-visible bar uses a quiet "toggle-on" background so it stays calm as the caret moves, while the menus use the bolder accent fill. The text-style (P/H) menu now **fills** the current level as a row (matching the other dropdowns) instead of showing a checkmark, and **greys out to "—"** where a heading can't apply — inside a table cell, a code block, on a selected image or formula, or while typing a callout's title — instead of misleadingly holding "P".
- **Proofreading switches read clearly when on**: the on-state of the Checks-menu toggles now uses the toolbar's accent fill (the same color as an active list row or the Full/Fixed width control) instead of a muted focus-border tint that looked washed out in some themes. The typography (**A**) menu is also reordered top-down to font size → content width → font family → Font settings.
- **Toolbar dropdown menus close instantly**: switching between adjacent dropdowns (e.g. Lists → Quote) no longer briefly shows both — a menu closes the moment the pointer leaves it, with a transparent bridge over the button→menu gap so the menu you're reaching for still holds open.
- **Typography (A) menu reordered**: the menu now reads Full Width / Fixed, then font size, then the font-family presets grouped with **Font settings** (no divider between the two), matching the top-down flow of width → size → family.
- **Leaner default toolbar — several less-used controls now ship hidden**: **Strikethrough, Inline Code, Task List, Callouts, Horizontal Rule, Math, and Clear Formatting** have moved out of the left group by default, leaving a shorter default bar (Format, Bold, Italic, Link, Bullet List, Ordered List, Code Block, Blockquote, Table, Image). Each hidden control stays reachable another way — Strikethrough via `~~…~~` or Mod+Shift+X, Inline Code via `` `…` `` or Mod+E, Task List via `- [ ] `, Callouts via the slash menu or `[!note] ` in a blockquote, inline math via `$…$`, horizontal rules via `---`, and Clear Formatting from the Command Palette and slash menu. Re-add any of them from Settings → Customize Toolbar (or `markdownWysiwyg.toolbar.items.*`). The math button's tooltip is now **Inline Math** (it inserts inline `$…$`; block math is a LaTeX code block, reachable via the code block's language switcher).
- **Customize-toolbar highlight no longer nudges the bar**: entering Customize Toolbar mode draws its zone outlines as a layout-neutral ring, so the toolbar buttons no longer shift by a pixel when you open (or leave) customize mode. The ring is a solid hairline rather than a dashed outline.
- **Every style check is on by default**: passive voice and negative parallelism, previously off because they can flag correct English ("was born", "is located", "not only X but also Y"), now ship on like the rest. If they get noisy, turn them off individually in the Checks menu or flip the master **Proofreading** switch off.
- **Grammar setting renamed**: `markdownWysiwyg.spellCheck.grammar` is now `markdownWysiwyg.grammarCheck.enabled` (grammar is a sibling of spelling, not a sub-option of it). The old key is deprecated but still read as a fallback, so an existing value keeps working — move it to the new key at your convenience.
- **Proofreading command consolidated**: the old "Toggle Style Check" command and the short-lived "Turn Proofreading On/Off" command are replaced by a single **Toggle Proofreading** command (⌘⌥⇧D / Ctrl+Alt+Shift+D) that flips the master gate. The granular Check Spelling / Grammar / Style commands remain.
- **Style-check explanations now tell you what to do**: each popup and hover explanation was rewritten to give the actual move instead of restating the label — e.g. passive voice reads "The doer is hidden or trailing — lead with who acts: 'mistakes were made' → 'we made mistakes'" rather than "Consider the active voice", and long sentences, clichés, wordiness, and the rest each carry concrete guidance.
- **Heading gutter shows the Markdown hashes**: the level marker beside each heading — both in the document and in the sticky heading that pins to the top on scroll — now renders the literal `#`, `##`, … `######` (dimmed, in the editor's monospace font) instead of `#H1`…`#H6`, and updates live as a heading's level changes.
- **Checks menu collapses its style options under "Check style"**: the dozen individual style checks (Fillers, Long sentences, Curly punctuation, …) are now nested — indented under the "Check style" master with a connecting rail — and shown only while Check Style is on. Turning Check Style off collapses the menu to just the three top-level toggles (Check spelling / Check grammar / Check style), instead of leaving a long list of greyed-out options for a feature that's off. (Previously the sub-checks stayed visible but dimmed; you can no longer pre-toggle a sub-check while its parent is off — enable Check Style first.) The slash menu is unchanged: it still surfaces only the three masters, so the two stay consistent.
- **Proofreading — one popup for every finding, style checks now interactive**: clicking any flagged span — spelling, grammar, *or* a style hit (filler, cliché, long sentence, em dash, …) — now opens the same popup, whereas style hits previously offered no click action at all. Each finding leads with a category chip ("Filler", "Long sentence") and its explanation, and carries its own actions: a one-click **Fix**/**Remove** where the correction is deterministic (delete a filler, normalize a curly quote or em dash to ASCII), plus **Ignore** (session) on every finding and **Add to dictionary** on spelling. When findings overlap — a filler word inside a long sentence — the popup **stacks** them, most-specific first, so each is reachable instead of only the topmost. A long sentence is no longer double-flagged: where the grammar engine's own long-sentence finding (with its word count) covers a sentence, the blunter style flag drops out; but because the grammar engine only flags very long sentences, moderately long ones still get the style flag, so coverage never regresses. Deterministic fixes are one click (delete a filler without leaving a double space, fold a curly quote or em dash to its ASCII form), and hovering a hit now shows a pointer cursor and reveals the span, signalling it's clickable.
- **Table of contents appears instantly on open**: when the rendered editor opens with the contents sidebar auto-shown, the panel is now already in place rather than sliding and fading in — the transition no longer draws attention to the switch into the editor. Showing or hiding the sidebar yourself still animates as before.
- **Frontmatter table is borderless**: the metadata table no longer draws an outer frame, rounded corners, or lines between fields — it reads as a plain key/value grid on the editor background.
- **Proofreading no longer blocks or interrupts opening a document**: spelling, grammar, and style underlines now settle in on their own a moment after the editor appears, instead of being computed as part of opening the file (which made opens slower) — and they never flash in as a reaction to your first click or keystroke. Proofreading is decoration only; it never delays the editor becoming ready to type in. When every check is turned off, it does nothing at all — no scan, and the grammar engine is never loaded — so a feature you've disabled costs you nothing.
- **Faster editor launch**: opening a Markdown file now loads far less up front. The KaTeX math stylesheet (~1.4 MB), the syntax-highlighting grammars (~66 languages), and the Mermaid diagram engine are no longer part of every launch — each loads on demand the first time the document actually needs it (math is rendered, a code block appears, a diagram is shown), and the round-trip-fidelity check that runs when a file opens no longer blocks the first paint. A document with no math, code, or diagrams now downloads and parses roughly a third of what it did before; documents that use those features load them the moment they appear, with no change to how anything looks or round-trips. Larger files feel the biggest opening speed-up.
- **Image alt text is an always-visible caption**: an image's alt text now shows as an editable caption directly under the image (italic, dimmed; revealed on selection when empty) instead of hiding behind a toolbar "ALT" button. Click the caption to edit — Enter or clicking away applies the change to the document, Escape reverts.
- **Image title shows as a hover tooltip and is editable**: the markdown title (`![alt](src "title")`) now surfaces the way published HTML treats it — hover the image to see it. The image toolbar carries the title as an always-visible second row: click to edit, Enter or click-away applies, Escape reverts.
- **Image path editing applies on blur**: the image toolbar's ✓/✗ confirm buttons are gone — the editor opens from a single file-name chip (name + pencil glyph) in the toolbar, applies on Enter or on clicking away, and Escape cancels, matching how link edits already work.
- **Table editing rebuilt**: tables now render with Google-Docs-style overlay chrome — row/column grips, hover insert bars, and drag-to-reorder — replacing the previous inline table UI.
- **Live theme following**: in the default `auto` mode the editor now recolors the instant you switch VS Code themes — including light ↔ dark and the high-contrast variants — and Mermaid diagrams re-render to match. Previously it froze its colors at open time and only picked up the new theme after the webview reloaded (closing and reopening the file, or toggling raw view and back). Pinned built-in themes and user `customThemes` (via `markdownWysiwyg.colorTheme`) keep their own palette by design and are unaffected.
- **Pinned/custom themes recolor accents too**: when you pin a theme or use `customThemes`, its palette now drives the editor's accent colors (callout/directive accents, proofreading underlines, syntax highlighting, links, focus rings) — previously a pinned theme only recolored a base subset and these accents stayed on VS Code's active-theme colors regardless.
- **Metadata panel — unambiguous delete and unified controls**: each field row's delete control is now a trash-can button to the left of the field name (revealed on row hover or keyboard focus) with a "Delete field" tooltip, so it can no longer be confused with the `×` that removes a single list chip. The collapse toggle and "Add field" button now share one style instead of two different bordered looks. Deleting the last field keeps the panel open (empty) so you can re-add a field or undo, rather than making the whole panel disappear.
- **Metadata panel keyboard and screen-reader support**: every control in the frontmatter panel (collapse toggle, Add field, per-row delete, chip add/remove) now activates with Enter/Space, the field cells expose text-box roles and labels, the delete button's accessible name includes the field it removes, and the collapse toggle reports `aria-expanded`.
- **One link editor for inserting and editing**: inserting a link (toolbar button, ⌘K/Ctrl+K, slash menu) now opens the *same* panel that appears when you hover an existing link — there's no longer a separate compact insert box that looked and behaved differently. Insert opens it anchored right at the selected text (not up by the toolbar) and keeps that text highlighted while you type, so you can see what the link applies to. It applies on Enter or when you click away, Escape cancels, and there are no ✓/✕ confirm buttons — the same apply-on-blur pattern as the image editors.
- **Link format is a dropdown**: the `markdown` / `[[wiki]]` choice in the link editor is now a labeled "Format" dropdown instead of a two-button toggle — it reads more clearly and is fully keyboard- and screen-reader-operable. The wikilink option is disabled for external targets (scheme URLs, `#anchors`).
- **Reference links stay faithful**: `[text][ref]` reference links and their `[ref]: url` definitions are kept visible and round-tripped instead of being silently inlined on edit.
- **Round-trip-lossy constructs pinned to saved bytes**: constructs the editor can't re-emit byte-for-byte are pinned to their saved bytes on write, so an edit elsewhere in the file never rewrites them.
- **Hard fork + English-only base**: the project was hard-forked and rebranded, all Chinese content was removed, English is now the source/base language across code, UI, and docs, and a CI guard blocks new non-ASCII (CJK) literals from regressing it.
- **Saving is fully native**: because the editor is backed by a real text document, saving now follows VS Code's built-in `files.autoSave` / `files.autoSaveDelay` (set `files.autoSave` to `afterDelay` to write automatically after you stop typing). The editor no longer runs its own save timer — the old one only ever acted where it was redundant with, or fought against, your `files.autoSave` choice. With the VS Code default (`off`), edits stay dirty (● in the tab) until you press Cmd+S, like any editor.

### Removed

- **Editor color-theme picker (`markdownWysiwyg.colorTheme` / `markdownWysiwyg.customThemes` settings, "Select Color Theme" & "Show Current Theme" commands)**: the editor now always follows your active VS Code color theme — including live theme switches and OS light/dark changes — with no separate per-editor theme to set. Pinning a different built-in theme, or defining a hand-written custom theme for the editor pane, is gone; it duplicated VS Code's own theme picker for a niche result and was the source of the "editor theme doesn't update" class of bugs. If you had set either setting, VS Code will flag it as unknown — remove it from your `settings.json`.
- **`markdownWysiwyg.autoSave` / `markdownWysiwyg.autoSaveDelay` settings**: removed in favor of VS Code's built-in `files.autoSave` / `files.autoSaveDelay`, which govern this editor natively now that it is text-document-backed (see "Saving is fully native" above).

- **`markdownWysiwyg.imageSelectionColor` setting**: selection accents no longer take a custom color — the image/divider selection border and table cell-range selection now follow the active theme's focus color (`focusBorder`), like every other selection indicator.
- **Image file rename**: the image toolbar's inline file-name field no longer renames the image file on disk — an unlabeled field silently renaming workspace assets (and breaking undo: reverting the document couldn't rename the file back) was the wrong tool for an editor. The toolbar's file name now opens the path editor, which changes *which* image the document renders without touching files.
- **Image server upload**: removed the `server` image-storage mode and all `imageServer*` settings (`imageServerUrl`, `imageServerFieldName`, `imageServerExtraParams`, `imageServerResponsePath`) along with the `imageStorage` toggle. Images are now always saved to the local workspace and are never uploaded off the machine — the editor no longer has any network egress path for document content.
- **Send to Claude**: removed the selection-toolbar "Send to Claude" button and the `Option+K` / `Alt+K` shortcut, along with the terminal/extension detection plumbing.
- **External-link confirmation dialog**: opening an external link no longer shows an extension-level confirm — VS Code's own trusted-domains prompt on `openExternal` already covers it, so the extension dialog only stacked a second question on top. The `javascript:`/`file:`/`command:` scheme blocklist stays.

### Fixed

- **Block selection no longer double-paints**: with blocks selected (Escape, marquee, Shift+arrows), the native blue text highlight leaked through under the block tint — two competing blues that read as a glitch, especially in light themes. The tint is the single selection visual now.
- **Gutter icons align with the first line everywhere**: markers on quotes, callouts, directives, code blocks, tables, and footnotes rode several pixels above their block's first visible line (anchored to the box corner instead of the content); every marker now centers on the first line and the whole gutter column is pixel-straight. Revealed markers are also a touch higher-contrast.
- **Callouts, container directives, and Notion asides have gutter grabbers**: hovering their body reveals the marker (it previously only appeared from the bare gutter strip), directives and asides had no marker at all, and a folded callout's grabber survives the collapse instead of vanishing with the body.
- **Folded sections move as one unit from the keyboard**: Escape on a collapsed heading selects the heading with its hidden content, Shift+↑/↓ absorb a collapsed section atomically (never silently selecting invisible blocks), and Alt+↑/↓ moves carry the hidden body along — previously the heading line could travel alone, stranding its collapsed content and re-folding whatever followed the drop. The selection tint also correctly appears over folded selections, and dragging a covered marker gets the same protection.
- **The sticky heading bar's fold chevron targets the right heading** after content above it shifts (e.g. an external file sync) — it previously acted on a stale position.
- **Tables get a gutter grabber too**: tables now show the table icon in the gutter (hover the table body to reveal it) with the actions-only block menu — duplicate, copy as markdown, move, delete — and drag-to-move, like every other block. Table conversions stay off the menu; the table's own grips and insert bars are unchanged.
- **Footnote markers reveal from the footnote itself**: hovering a footnote definition's content now reveals its gutter marker — previously only the empty gutter strip to its left did (same fix covers the table marker).
- **Item grabber spacing evened out**: the gap between a list item's grabber icon and its bullet/number/checkbox is now visually consistent across list flavors — bullets no longer show a much larger gap than numbered items (the offsets are tuned to each flavor's marker ink, not the box).
- **Tab indents just the current list item**: pressing Tab on an item that has nested sub-items now indents that item alone — its children keep their level and become its siblings (previously the whole subtree was dragged one level deeper). Shift-Tab remains the exact inverse.
- **Alt+↑/↓ in a list moves the item, not the list**: with the caret in a list item (at any nesting depth), the move-block keys now reorder that item among its siblings; previously they moved the entire list as one block.
- **Cmd+A escalation works inside VS Code**: the chord was reaching VS Code's own webview select-all, whose full-document selection immediately stomped the ladder — it's now claimed by the editor while focus is in the document (overlay inputs keep native select-all).
- **Auto-scroll ramps instead of sprinting**: dragging a block or marquee near the viewport edge now accelerates with pointer depth — barely entering the zone creeps, the edge is brisk, and pushing past the edge reaches full speed — instead of jumping straight to maximum at the zone's first pixel.

- **The block menu no longer floats away from its marker**: scrolling with a gutter menu open now keeps it glued to the marker that opened it (it used to stay at fixed screen coordinates while the document scrolled underneath). The menu also closes when the document changes under it (typing, external file sync) and when the window loses focus — and every menu action double-checks it's still acting on the block it was opened for, so a stale menu can never delete or convert the wrong content.
- **Collapsed sections stay collapsed when moved — and never collapse a neighbor**: moving or dragging a collapsed heading section now carries the collapse with it to the destination. Previously the fold could silently transfer to whatever heading landed at the old position, hiding the wrong section; deleting a collapsed heading had the same defect. Inserting content directly above a collapsed heading (duplicating the block above it, dropping a section there) also no longer expands it or shifts the collapse onto the inserted block.
- **Converting a titled callout keeps the title**: turning `> [!TIP] My Title` into a blockquote, paragraph, or list now carries "My Title" along as leading prose / the first item — it used to vanish silently.
- **The block menu respects small windows**: when the menu doesn't fit below or above its marker it now picks the larger side, stays out of the toolbar's band, and scrolls internally — it used to clamp to the top of the window, sliding under the toolbar and covering its own block. It also re-anchors when the editor pane is resized, and sits above the sticky heading.
- **Copy as Markdown copies the block you asked for**: with text selected in one block, using "Copy as Markdown" from another block's gutter menu used to copy the selection instead of the block.
- **Copy Link targets the right duplicate heading**: copying a link from the second `## Setup` now yields `#setup-1` (matching how anchors actually resolve, here and on GitHub) instead of a link to the first duplicate. Internal anchor resolution also no longer trips over the gutter's `##` glyphs when headings share text.
- **Drag sessions can't get stuck**: releasing the mouse outside the window, or the window losing focus mid-drag, now ends the drag cleanly — and an Escape-canceled drag no longer swallows the marker's next click.
- **No more "P" marker on image and raw-HTML blocks**: standalone images and raw HTML blocks (which parse as paragraphs wrapping a single inline element) no longer show the hover-revealed `P` gutter marker or its heading retype menu — those are text affordances and never made sense there. Paragraphs mixing text with inline images keep their marker.
- **Choosing a heading while in a list now works**: with the cursor inside a bullet/ordered/task list, picking **H1–H6** from the text-style (P) dropdown, the command palette, or the slash menu previously did nothing — a list item can't hold a heading. That line is now lifted out of the list and promoted to a heading (splitting the surrounding list, and climbing all the way out of nested lists), matching Notion/Obsidian. Picking **Paragraph** on a list line stays a no-op (it's already a paragraph — list membership is controlled by the Lists dropdown).
- **"Editor font" now clears a serif/sans/mono preset instead of leaving the content stuck**: switching the content font back to **Editor font** (which inherits VS Code's `editor.fontFamily`) had no effect when the document opened on a non-editor preset — the picker's checkmark moved but the prose stayed in the previous preset (e.g. serif) until a window reload. The initial preset is now applied so that selecting **Editor font** takes effect live.
- **Checks menu no longer flickers when toggling proofreading on/off**: "Turn off/on proofreading" writes three settings at once (spelling, grammar, style), and each write used to echo a half-applied state back to the menu, so the checkmarks visibly flipped through intermediate states before settling. Those echoes are now coalesced into a single update of the final state.
- **Grammar check no longer flags inline code as run-on spaces**: a sentence with inline `code`, an image, or an escaped table pipe could trigger a bogus grammar finding like "There are 28 spaces where there should be only one" whose underline landed on the code itself — the masked content was being handed to the grammar engine as a run of spaces. Those spans are now invisible to the engine, so the false finding is gone.
- **Clicking a link no longer strands the open link editor**: with the link editor open (inserting or editing a link), clicking on any existing link elsewhere in the document now applies your edit and closes the editor — matching the click-away behavior everywhere else. Previously the click did nothing and the editor stayed stuck open until you pressed Escape or clicked a non-link spot.
- **Command palette entries all carry the editor's name**: every command this extension contributes now shows under the **"WYSIWYG Markdown Editor:"** prefix in the command palette. Previously the mode-switch, debug, style-check, and go-to-symbol commands showed bare — e.g. a plain "Select Color Theme" that sat next to VS Code's own "Preferences: Color Theme" with nothing marking it as this editor's.
- **Image toolbar viewport flip**: with an image selected near the top of the viewport, the floating toolbar now measures its own height when deciding to flip below the image, so the two-row toolbar (controls + title) never clips off-screen.
- **Hidden toolbar no longer leaves phantom toolbar space**: with the toolbar hidden (`markdownWysiwyg.toolbar.visible` off), the table-of-contents panel kept a toolbar-sized gap above its header — and showing the bar again could bury the header underneath it. The panel now tracks the toolbar's real state and slides in sync with it, and every scroll target that reserved room for the bar (clicking a TOC entry, find-bar match navigation, footnote jumps, in-page anchor links, view-source line sync) now lands correctly instead of ~40px off when the bar is hidden.
- **Link fragments survive modifier-click**: the click handler stripped `#…` from the href before messaging the host, which broke `file.md#27` line navigation and dropped anchors from external URLs.
- **Find bar tab order**: Tab in the Find field jumps straight to Replace when the replace row is open (Shift+Tab returns), matching VS Code's find widget, and the cycle wraps after Replace All back to the navigation buttons so every control stays keyboard-reachable; Escape still exits to the editor.
- **Inline math in a code block no longer crashes**: with the caret inside a code block (or a block node selected), the math toolbar button threw an internal error; the command now refuses cleanly where inline math cannot live.
- **Screen-reader names for toolbar buttons**: every toolbar button and dropdown trigger (Format, Font, Checks, Settings, overflow, Debug) now carries an `aria-label`; icon-only buttons derive it from their tooltip automatically, minus the shortcut hint.
- **Mark input rule near the end of the document**: applying a mark input rule (`==x==`, `**x**`) very close to the end of the document could silently fail to apply because the affected range was computed past the document's end. The range is now clamped so the mark always applies.
- **Tight lists stay tight on save**: a tight list (no blank lines between items) is no longer loosened to blank-line-separated items when a file is opened and saved — including lists nested inside blockquotes, directives, and callouts. The loosening was previously masked at runtime by round-trip protection, so this hardens fidelity by construction rather than changing what you saw on screen; genuinely loose lists still save loose.
- **Undo reaches metadata edits**: ⌘Z/Ctrl+Z now undoes changes made in the frontmatter panel — editing a value, adding a field, or deleting one — and ⌘⇧Z/Ctrl+Y redoes them. VS Code intercepts the shortcut before the panel could ever see it, so metadata edits previously could not be undone from the editor at all; the panel now keeps its own history and a chord anywhere over the panel (including blank space) walks it.
- **Shift+Tab in the metadata panel**: pressing Shift+Tab in a field's value cell now moves back to that field's name (and from a name back to the previous field's value), instead of spawning a stray new field row.
- **Link editor opens next to the link**: the link editor now positions itself directly under the text it applies to. Previously, focusing its input could scroll the view first, landing the panel well below the link (or off near the bottom of a long document).
- **Switching between rendered and Raw Markdown no longer spawns duplicate tabs**: toggling a document with unsaved edits between the WYSIWYG editor and Raw Markdown now shows VS Code's normal Save / Don't Save / Cancel prompt and acts on the answer — Save or Don't Save switches modes, and **Cancel truly cancels** (you stay in the current mode). Previously, Cancel still opened the other mode in a *second* tab, leaving the same file open twice (once rendered, once raw).

  - **Native undo/redo**: Cmd/Ctrl+Z and Shift+Cmd/Ctrl+Z now use VS Code's document history and stay in sync with edits made in a side-by-side text view.

  - **Git integration**: because edits flow through the document, staging, diffing, discarding changes, and `git checkout` update the editor live.
- **Cursor-preserving inbound sync**: external changes (a side-by-side text edit, undo/redo, git operations, hot-exit restore) are applied to the editor as a minimal ProseMirror diff rather than a full rebuild, so the caret and selection survive edits made elsewhere in the document. ### Fixed

- **Formatted links no longer split**: a link whose text contains bold/italic/code (e.g. ``[**bold** and `code`](url)``) previously shattered into several adjacent links when its line was edited; it now serializes as a single link. Also root-fixes the related strong-around-link splitting.
- **Line breaks inside table cells**: `<br>` in a GFM table cell is preserved instead of being silently flattened to a space on save; Shift+Enter inserts one.
- **Remote/SSH/WSL/Codespaces**, **Cmd+B/Cmd+I no longer leak to VS Code**, **undo inside overlay inputs**, and **lossless frontmatter editing** (earlier Phase 0 fidelity fixes).
